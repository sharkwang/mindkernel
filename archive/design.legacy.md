# Memory v2 设计草案

## 问题陈述
现有记忆体系的主要问题：
1. 写入不稳定：重要讨论未必落盘
2. 信息层次混杂：日记/长期记忆/配置知识边界不清
3. 可追溯性弱：难以定位来源会话
4. 缺乏闭环：没有固定的提炼与清理节奏

## 设计原则
- 先可用，再完美
- 可追溯优先于“看起来聪明”
- 自动采集 + 人工确认并重
- 可回归测试（能衡量改进）

## 工作流（草案）
1. Capture：从会话生成候选记忆
2. Curate：去重、合并、打标签、确认置信度
3. Publish：写入长期记忆与索引
4. Review：周期复盘，淘汰过期记忆

## 待定问题
- 候选记忆触发时机（每轮/每会话/仅关键事件）
- 置信度模型与人工确认阈值
- MEMORY.md 与结构化存储如何双写
